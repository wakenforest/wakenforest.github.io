包（Package）

包声明

包声明的语法：

  • 包用来组织各式各样的声明，使其可以被许多设计共享使用。包存储在library中。一个包被分为包声明（强制的）和包主体（可选）。

  • 共享的类型包括，子类型，常量，信号，文件，断言，元件，属性和组。

  • 使用use子句，包中声明的项就可以在其他设计单元中可见。

包声明的例子

  • 包声明可以包含子程序（函数或过程）声明；但子程序的主体不能出现在包声明中，必须在包主体内。

  • 如果声明中包含子程序声明或延迟常量，则包主体需要与包声明一致。

  • 只有在包声明中定义的对象对外可见。

包主体（Package Body）

包主体定义了包的子程序，以及包中定义的延迟常量值。

包主体的语法

包主体中包含完整的子程序主体，以及延迟常量值。其他声明也可以出现在包主体中，但是仅对包内部可用。

包主体的例子

  • 除了延迟常数值和子程序主体外的声明对包主体外不可见。

  • 每个包仅有一个主体。

库（Library）

Library子句在本地环境下，为设计库定义一个逻辑名。

库的语法(引入库)：

Library语句为设计库定义一个逻辑名，可以被设计单元使用。

VHDL库常被作为主机文件系统的一个路径来实现。库名对应文件路径。

有两个预定的库，被隐式使用：std和work。Std库包含包Standard和Textio。

Work库是工作库，存储所有用户创建的分析的设计单元。所有用户指定的包都存储在工作库Work中。

库的例子：

  • 在主设计单元中使用library语句指定的库，对其相关联的次级单元可见。

  • Std库没有必要特别指定。所有的库包被自动包含在每个设计单元中。

使用语句（Use）

预定义库

Standard Package

该packge包括了基本类型，子类型，函数和操作符等的定义。

每个VHDL代码都隐式调用了Standard，并不需要显示调用。

TEXTIO Package

Textio包括了支持格式化I/O操作的数据类型和函数。
允许VHDL读取ASCII text文件，或者写这些文件。

可以看到有3种类型：LINE,TEXT和SIDE，以及很多的读写函数。

Std_logic_11644 Package

用于定义VHDL中的数字逻辑值，包含了std_logic和std_logic_vector。它也包含了VHDL函数，用于处理三态冲突、定义逻辑运算符和与其它标准类型相互转换的函数。

Numeric_Std Package

用于提供IEEE 1164 std_logic中的值的更清晰的定义。
